## Debug 等級

| 等級 | 工具 | 適用情境 |
|------|------|---------|
| L1 | UART Log | 最基本，任何環境 |
| L2 | MSC/HID Debug Channel | 不需要額外工具 |
| L3 | ITM/SWO Trace | 即時追蹤，不 halt MCU |
| L4 | GDB + OpenOCD | 全功能 Debug |

## Debug 心法

### 1. 二分搜尋法
```
懷疑區間 → 分割 → 縮小範圍 → 找到問題
```

### 2. 單一變數法
```
一次只改一個變數，觀察行為變化
```

### 3. 最小重現法
```
找到能穩定重現問題的最簡單步驟
```

## 通用流程

1. **重現問題** — 找到穩定重現步驟
2. **假設** — 猜測可能原因
3. **驗證** — 用 Log/斷點確認
4. **修復** — 最小變更原則
5. **確認** — 重新測試

## 常見錯誤模式

| 模式 | 說明 |
|------|------|
| Off-by-one | 陣列索引多1或少1 |
| Race condition | 時序相依的錯誤 |
| Memory leak | 記憶體未釋放 |
| Buffer overflow | 寫入超出邊界 |
